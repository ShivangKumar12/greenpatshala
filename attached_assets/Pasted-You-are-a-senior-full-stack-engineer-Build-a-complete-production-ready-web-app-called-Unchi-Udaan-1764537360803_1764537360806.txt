2. Roles & access control

Implement 3 roles: admin, instructor, user.

Both admin and instructor can:

Create / edit / delete courses

Upload and manage study materials

Create / edit quizzes & questions

Create / edit current affairs

Admin only can additionally:

Manage all users (view list, change roles, enable/disable accounts)

Manage job posts (CRUD)

Manage coupons

Users can:

View and purchase courses/quizzes/materials

Attempt quizzes

View study materials & current affairs

View job portal

Implement role-based middleware on backend routes and ProtectedRoute / RoleProtectedRoute on frontend.

3. Auth, OTP, JWT, Google login

Registration:

User registers with name, email, password.

Hash passwords with bcrypt.

On register, send OTP to email using SMTP (Nodemailer). SMTP config (host, port, user, pass) comes from .env.

Store OTP with expiry in DB (or separate verification table/field).

Verify OTP route to activate account.

Login:

Local login (email + password) → issue JWT (access token).

Store JWT in HTTP-only cookie or in memory (plus refresh token if you want) – choose secure pattern and document it.

Implement Google login using Google OAuth2:

Use Passport.js or a clean library.

On Google login success, create user if not exists, then issue JWT.

Include /auth/me endpoint to fetch current user and role.

4. Payments: Razorpay integration

Integrate Razorpay for payments (test mode).

Keep these env vars in .env: RAZORPAY_KEY_ID, RAZORPAY_KEY_SECRET.

Allow Buy Now for:

Courses

Quizzes

Study materials

Any other paid items

Features:

Each item has:

original price

discount price

optional coupon support

Implement coupon codes:

Admin creates coupons (code, discount type %, flat, expiry, usage limit).

Validate coupon on backend before creating Razorpay order.

Store each successful payment in a Payments table with:

user_id, item_type, item_id, amount, status, Razorpay order/payment IDs, timestamp.

After payment success:

Mark user as enrolled / purchased in respective tables.

Allow user access to content/quiz accordingly.

5. Quizzes (random, time-based, PDFs)

Quiz features:

Admin/instructor can:

Create quiz

Add multiple questions (MCQ with 4 options, correct option, marks, etc.)

Mark quiz as paid or free

Set time duration for quiz (e.g., 15 min, 30 min)

For users:

Start quiz → backend sends randomized questions each time (shuffle).

Enforce time-based quiz:

On start, backend returns start time + duration.

Frontend shows countdown timer.

On expiry, auto-submit answers.

Store quiz attempts:

user_id, quiz_id, score, percentage, number of correct/incorrect, started_at, completed_at, etc.

PDF export:

Backend utility pdfGenerator.js to generate:

Quiz questions PDF

For admin/instructor: include quiz title, all questions, options, mark per question.

Quiz result PDF per user

Includes user name, quiz title, score, percentage, date/time, maybe detailed question-wise report.

Use a Node PDF library (like pdfkit or pdfmake). Provide API endpoints:

GET /quizzes/:id/export-questions → returns PDF

GET /quizzes/:id/attempts/:attemptId/export-result → returns PDF
